# Python Audit Skill

## 目标
- 聚焦 Python 项目的静态审计
- 识别安全风险、异常处理缺陷、并发与性能隐患

## 审计优先级
- 优先检查命令执行、反序列化、路径穿越、模板注入与不安全配置加载
- 优先检查异常吞噬、宽泛 `except`、错误重试失控与关键流程无回滚
- 优先检查线程与异步混用带来的竞态、阻塞调用、死锁与超时缺失
- 优先检查敏感信息泄漏、日志明文输出、凭证硬编码与权限越界

## 代码检查要点
- 查找 `subprocess`、`os.system`、`eval/exec` 调用与参数来源
- 查找 `yaml.load`、`pickle`、`marshal`、动态导入等危险入口
- 查找文件读写与路径拼接逻辑，确认工作区边界与规范化处理
- 查找网络调用与外部依赖，确认超时、重试上限、错误分级
- 查找类型与接口契约不一致，确认关键参数校验和返回值稳定性

## 输出要求
- 每个结论必须包含证据位置（文件路径、行号、符号）
- 修复建议必须可直接落地，优先给出最小改动方案
