# JavaScript / TypeScript Audit Skill

## 目标
- 聚焦 JavaScript 与 TypeScript 项目的静态审计
- 识别高风险缺陷、类型与异步问题、构建与运行时隐患

## 审计优先级
- 优先检查鉴权、输入校验、命令执行、路径拼接、反序列化与 XSS 注入链路
- 优先检查 Promise 未处理异常、并发竞态、资源泄漏、无限重试与超时缺失
- 优先检查 TypeScript `any` 滥用、类型断言掩盖错误、接口与实现不一致
- 优先检查 Node.js 服务端对外接口、文件系统访问、子进程调用与环境变量使用

## 代码检查要点
- 查找 `eval`、`new Function`、不安全动态导入、危险模板拼接
- 查找 `child_process` 执行入口，验证参数是否最小权限与严格白名单
- 查找 `fs` 读写路径来源，确认路径规范化与越权访问防护
- 查找 `fetch/axios` 调用，确认超时、重试、错误分级与敏感信息脱敏
- 查找依赖管理风险，关注锁文件缺失、宽版本范围与高危依赖调用

## 输出要求
- 每个结论必须包含证据位置（文件路径、行号、符号）
- 修复建议必须可直接落地，优先给出最小改动方案
